# 가상메모리
가상메모리를 사용한다면 당장 실행에 필요한 부분만 실제 메모리에 올려서 실행하게 된다.

즉 **프로그램 전체가 아닌 필요한 일부분만을 실제 메모리에 올리는 기법.**

장점
- 물리 메모리의 여유공간이 얼마나 되는지 시작 전에 고민할 필요가 없다
- 더 많은 프로그램들을 동시에 메모리에 올려 작업할 수 있다.
- 한번에 올리는 소스 양이 적어지기 때문에 Disk와 메모리간 I/O 작업 속도가 빨라진다.

가상 메모리는 프로세스의 물리 메모리와 논리 메모리를 분리하기 위해 생긴 개념.
논리 메모리가 물리메모리보다 커지는 것을 가능하게 한다.

# Paging

프로세스에게 메모리를 연속적으로 할당해주지 않는 방법.

프로세스의 논리적 메모리를 조각으로 나누어 그에 대응하는 물리 메모리공간 조각이 어디에 위치하는지 page table에 저장한다.
논리적 메모리의 조각을 page라고 하고 물리적 메모리의 조각을 Frame이라고 한다. page의 크기와 Frame의 크기는 같다.

# 페이지 교체
페이지 교체의 동작 원리
1. 디스크에서 적재가 필요한 페이지의 위치를 알아낸다
2. 메모리 공간에서 빈 페이지의 프레임을 찾아낸다
  - 이때 빈프레임이 없다면 페이지 교체 알고리즘을 통해 희생될 프레임을 선정
3. 새롭게 비워진 프레임에 새 페이지를 읽어오고 프레임 테이블을 수정한다.
4. 사용자 프로세스를 재시작

페이지 교체 알고리즘
- 선입선출 페이지 교체(FIFO Page Replacement)
  - 가장 오래된 페이지를 교체하는 알고리즘
- 최적 페이지 교체(Optimal Page Replacement)
  - 앞으로 가장 오랫동안 사용하지 않을 페이지를 찾아 교체하는 알고리즘
  - 프레임 수가 고정된 경우 가장 낮은 페이지 부재율을 보장한다.
- LRU 페이지 교체(Least Recently Used Page Replacement)
  - 가장 오랜 시간 **사용되지 않은** 페이즈를 교체
