# CI / CD란?
애플리케이션 개발 단계를 **자동화**하여 사용자에게 애플리케이션을 보다 짧은 주기로 제공하는 방법이다.

CI는 Continuous Integration, 즉 지속적 통합을 의미

변경사항이 정기적으로 빌드, 테스트되어 공유 리포지토리에 통합된다.
- 여러명의 개발자가 동시에 작업할 때의 충돌 문제를 해결

CD란 두가지 뜻이 있는데
1. Continuous Delivery 지속적인 서비스 제공
2. Continuous Deployment 지속적 배포


1. 지속적인 제공 (Continuous Delivery)
   - 변경사항이 버그 테스트를 거쳐 자동으로 **리포지토리에 업로드**되는 것.
2. 지속적 배포 (Continuous Deployment)
   - 변경사항을 리포지토리에서 고객이 사용 가능한 **프로덕션 환경까지 자동으로 릴리스**하는것.

> 대부분의 기업은 CI를 먼저 추가한 다음 **클라우드 네이티브 애플리케이션**의 일부로서 배포 및 개발 자동화를 구현해 나간다.

# +) 클라우드 네이티브 애플리케이션이란?
- 클라우드 컴퓨팅 기법과 기술을 기반으로 애플리케이션을 구축, 실행, 개선하는 방법
- 신규 애플리케이션 구축, 기존 애플리케이션 최적화, 모든 환경의 연결 작업을 가속화하는 애플리케이션

장점
- 다양한 환경 전반에 분산된 컴퓨팅 리소스를 갖춘 기업의 애플리케이션 개발 속도를 높일 수 있다는것.

// 기존 모놀리식 아키텍처는 규모가 크고 복잡하고 관리와 유지보수가 어려웠다.

## 클라우드 네이티브를 위한 네가지 기술
- DevOps
  - Dev(개발)+Ops(운영)의 합성어
  - 개발, 운영, 품질 엔지니어링, 보안 등 서로 단절되었던 역할들이 서로 조율하고 협업하여 더욱 안정적이고 뛰어난 제품을 생산하도록 하는 문화, 철학, 방식 및 도구의 집합.
- 마이크로서비스
  - 애플리케이션을 상호독립적인 최소구성요소로 분할
  - 세분화, 경량화하여 다수의 애플리케이션 간에 유사한 프로세스를 공유(엔터프라이즈 통합)에 의미를 둠
- CONTINUOUS DELIVERY
- 컨테이너

# +) 서버리스 아키텍처
> 애플리케이션 개발자가 서버를 프로비저닝하거나 애플리케이션의 확장을 관리할 필요가 없는 클라우드 컴퓨팅 모델을 말한다.
>
> 대신 클라우드 제공업체가 일상적인 테스크를 추상화하므로 개발자는 전통적인 모델보다 빠르게 코드를 프로덕션 환경으로 푸시할 수 있다.

클라우드 제공업체가 서버를 관리, 실행하여 요청이나 특정 이벤트가 있을 때 클라우드의 서버를 이용하거나 서비스할 어플리케이션을 동작시킨다.
따라서 인프라에 대한 별도 준비 없이, 자동 스케일링 방식으로 시시각각 변하는 자원 수요를 지원한다.
**서버 사양이 개발에 영향을 주지 않기 때문에 서버리스**라고 불리며
FaaS(Function as a Service), 백엔드 시스템을 보이지 않는 서비스로 추상화하였기 때문에 BaaS(Backand a Service)라고도 한다.

장점
- 개발 생산성을 높인다
  - 서버 프로비저닝 및 관리 불필요 -> 개발에 집중
- 운영 비용을 줄인다
  - 클라우드 기반 컴퓨팅 시간에 대한 비용을 지불하기 때문에 자체 서버 운영보다 비용 절감

단점
- 벤더 종속 문제
  - 자사 구성요소가 상호작용하는 방법을 엄격히 제한해 사용자 시스템의 유연성과 커스터마이징 수준에 영향을 줄 수 있다.
