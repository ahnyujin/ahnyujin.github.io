# 상속(inheritance)
- 정의
  - 기존 클래스를 재사용하여 새로운 클래스를 작성하는 것
  - 조상클래스를 extend
- 장점
  - 코드의 **재사용성**을 높이고 코드의 **중복을 제거**하여 프로그램의 생산성과 **유지보수** up

상속관계 이외에도 클래스를 재사용하는 방법 : 포함(Composition)관계
```
상속관계 '~은 ~이다.(is-a)'
포함관계 '~은 ~을 가지고 있다.(has-a)'
```
[자바는 다중상속이 불가능하다.](https://ahnyujin.github.io/posts/single-inheritance)
# 오버라이딩(overriding)
조상으로부터 상속받은 메서드의 내용을 변경하는 것.
- 메서드 선언부는 조상것과 일치해야한다. 다만 접근제어자와 예외는 변경될 수도 있다.
1. 접근 제어자를 조상 클래스보다 같거나 큰 범위로 변경가능
2. 예외는 조상클래스보다 많이 선언할 수 없다.

# 제어자 
제어자(modifier)는 클래스, 변수 또는 메서드의 선언부에 함께 사용되어 부가적인 의미를 부여
- 접근 제어자 - public, protected, default, private
- 그 외 - static, final, abstract, native, transient, synchronized, volatile, strictfp

# 다형성
다형성이란?
- 객체지향개념에서 '여러가지 형태를 가질 수 있는 능력'
- 자바에서는 조상클래스 타입의 참조변수로 자손클래스의 인스턴스를 참조할 수 있도록 하였다.

## 그렇다면 인스턴스를 같은 타입의 참조변수로 참고하는것과, 조상클래스 타입의 참조변수로 참조하는 것은 어떠한 차이가 있을까?
> 참조변수의 타입에 따라 사용할 수 있는 멤버의 개수가 달라진다.

부모타입의 참조변수로는 자식 인스턴스 중에서 부모클래스의 멤버들(상속받은 멤버포함)만 사용할 수 있다.

자손타입의 참조변수로 조상타입의 인스턴스를 참조하는것은 불가능. // 컴파일 에러

## 참조변수의 형변환
서로 상속관계에 있는 클래스 사이에서는 형변환이 가능하다.
자손타입에서 조상타입으로의 형변환(UP-casting)은 형변환 생략이 가능하나,
조상타입에서 자손타입으로의 형변환(Down-casting)은 형변환 생략불가하다.

# 추상클래스
추상클래스 자체로는 인스턴스를 생성할 수 없다. 상속을 통해 자손클래스에 의해서만 완성될 수 있다.
추상 메서드
선언부만 작성하고 구현부는 작성하지 않은 채로 남겨둔 것.
이유 : 상속받는 클래스에 따라 메서드의 내용이 달라질 수 있기 때문에 선언부만 작성하고, 주석을 덧붙여 어떤 기능을 수행할 목적으로 작성되었는지 알려주고, 실제 내용은 상속받는 클래스에서 구현하도록 함.
추상클래스를 상속받는 자손 클래스는 조상인 추상클래스의 추상 메서드를 모두 구현해주어야 한다. 그렇지 않으면 자손클래스 역시 추상클래스로 지정해야함.

추상화 : 기존 클래스의 **공통요소**를 뽑아서 조상 클래스를 만드는 것 <-> 구체화 : 상속 (세분화)

# 인터페이스
일종의 추상클래스. 추상클래스보다 추상화 정도가 높아서 일반 메서드 또는 멤버변수를 구성원으로 가질 수 없다.
오직 추상메서드와 상수만을 멤버로 가질 수 있다.
- 모든 멤버변수는 public static final 이어야 하며, 이를 생략할 수 있다.
- 모든 메서드는 public abstract 이어야 하며, 이를 생략할 수 있다. 단, static 메서드와 디폴트 메서드는 예외(JDK1.8부터)

인터페이스는 인터페이스로부터만 상속이 가능하며, 클래스와 달리 다중상속이 가능하다.

인터페이스를 이용한 다형성
- 인터페이스 타입의 참조변수로 이를 구현한 클래스의 인스턴스를 참조할 수 있다.

따라서 메서드의 매개변수 타입으로도 사용 가능.
- 메서드의 리턴타입을 인터페이스 타입으로 지정할수도 있다.
- **리턴타입이 인터페이스라는 것은 메서드가 해당 인터페이스를 구현한 클래스의 인스턴스를 반환한다는 것.**

# 내부 클래스
클래스 내부에 선언된 클래스

장점
- 내부클래스에서 외부 클래스의 멤버를 쉽게 접근할 수 있다.
- 외부에서는 불필요한 클래스를 감춤으로써 코드의 복잡성을 줄일 수 있다.

A의 내부 클래스 B
```java
class A { // 외부 클래스
    ...
    class B { // 내부 클래스
        ...
    }
    ...
}
```
A와 B는 긴밀한 관계일 것. 또한 B는 A 이외의 다른 클래스에서 잘 사용되지 않을것

# 익명 클래스
클래스의 선언과 객체 생성을 동시에 하기 때문에 단 한번만 사용될 수 있고 오직 하나의 객체만을 생성할 수 있는 일회용 클래스이다.
