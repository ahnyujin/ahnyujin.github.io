# 프록시란?

- 클라이언트와 서버간의 중계 서버로, 통신을 대리 수행하는 서버
- 캐시, 보안, 트래픽 분산 등 여러 장점을 가질 수 있음

## 포워드 프록시 - 클라이언트와 인터넷 사이에 위치시킴

특징
1. 캐싱
    - 전송 시간 절약
    - 불필요한 외부 전송 x
    - 외부 요청 감소 → 네트워크 병목현상 방지
2. 익명성
    - 서버가 응답 받은 요청을 누가 보냈는지 알지 못하게 함 // 서버가 받은 요청 ip는 프록시의 ip이기 때문

## 리버스 프록시 - 인터넷과 서버 사이에 위치시킴

특징
1. 캐싱
2. 보안
    - 클라이언트는 리버스 프록시를 실제 서버라고 생각하여 요청// 실제 서버의 ip가 노출되지 않음
3. 로드 밸런싱
    - 부하분산 - 해야할 작업을 나누어 서버의 부하를 분산시키는 것

로드 밸런서란? 여러대의 서버가 분산을 처리할 수 있도록 요청을 나누어주는 서비스

이외에도 HTTPS 인증서 관리를 프록시서버에서 하고 뒤에 동작하는 서버는 HTTP로 서비스한다면 서버를 실행할때마다 인증서를 관리하지 않아도 되는 장점이 있다.
